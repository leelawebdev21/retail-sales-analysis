# Retail Sales Analysis Dashboard

A beginner-friendly data analysis project using **SQLite** and **Python** to explore sales trends, product performance, and regional insights from a synthetic superstore dataset.

![Monthly Trend](charts/02_monthly_trend.png)

---

## Project Overview

| Item | Detail |
|------|--------|
| Dataset | Synthetic Superstore Sales — 10,000 rows, 2021–2023 |
| Tools | Python, SQLite, pandas, matplotlib, seaborn |
| Focus | Sales trends, regional analysis, product ranking, discount impact |

---

## Key Findings

- **Total Revenue:** $21.4M across 10,000 orders (2021–2023)
- **Profit Margin:** Consistently ~20% across all years
- **Top Region:** West drives the highest revenue; profit share is balanced across regions
- **Technology** is the highest-revenue category; Office Supplies has the most sub-categories
- **Discount impact:** Orders with 40%+ discounts show negative profit margins — over-discounting hurts profitability
- **Standard Class** shipping is used in over 50% of orders

## How to Run

### 1. Clone the repo
```bash
git clone https://github.com/YOUR_USERNAME/retail-sales-analysis.git
cd retail-sales-analysis
```

### 2. Install dependencies
```bash
pip install pandas numpy matplotlib seaborn jupyter
```

### 3. Generate the dataset & database
```bash
python generate_dataset.py
```

### 4. Open the notebook
```bash
jupyter notebook retail_sales_analysis.ipynb
```

Run all cells from top to bottom. Charts will be saved to the `charts/` folder.

## SQL Highlights

All analysis is powered by SQL queries run via Python's `sqlite3` module. Example:

```sql
-- Monthly revenue trend
SELECT
    STRFTIME('%Y-%m', order_date) AS month,
    ROUND(SUM(sales), 2)          AS monthly_revenue,
    ROUND(SUM(profit), 2)         AS monthly_profit
FROM sales
GROUP BY month
ORDER BY month;
```

See [`queries.sql`](queries.sql) for all 10 queries.
